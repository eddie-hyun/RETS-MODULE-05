## 3. SCR-001 — 대시보드

### SRS 추적: DEV_FR_005_01, DEV_FR_005_01_01

### 3.1. 화면 목적

모듈 전체 진행 현황을 한눈에 파악하고, 주요 작업 섹션으로 빠르게 이동한다.

### 3.2. 화면 구조

```
┌────────────────────────────────────────────────────────────────┐
│ DASHBOARD                                         [← 새로고침] │
├──────────────┬──────────────┬──────────────┬───────────────────┤
│ 📋 총 요구사항 │ 🧪 총 TC 수   │ 📊 커버리지   │ ⚠ 변경 요구사항   │
│   [N] 건      │   [N] 건     │   [N] %      │   [N] 건          │
│ (파란 카드)   │ (초록 카드)  │ (오렌지 카드) │ (빨간 카드, 있을때)│
├──────────────┴──────────────┴──────────────┴───────────────────┤
│ 최근 활동                                                       │
│  • [timestamp] DEV_FR_001_01 — TC 3건 생성됨                   │
│  • [timestamp] DEV_FR_002 — 변경 플래그 설정됨                  │
│                                                                │
├────────────────────────────────────────────────────────────────┤
│ 빠른 실행                                                       │
│  [+ JSON 불러오기]  [▶ TC 생성]  [📤 CSV 출력]                  │
└────────────────────────────────────────────────────────────────┘
```

### 3.3. UI 요소 상세

| 요소 | 타입 | 설명 |
|------|------|------|
| 총 요구사항 카드 | StatCard | 로드된 DEV-REQ 건수, 클릭 시 SCR-002 이동 |
| 총 TC 수 카드 | StatCard | 등록된 TC 건수, 클릭 시 SCR-003 이동 |
| 커버리지 카드 | StatCard | TC 보유 요구사항 수 / 전체 × 100%, 배경색 #F97316 |
| 변경 요구사항 카드 | StatCard (Warning) | 변경 플래그 수, 0이면 숨김 또는 회색, 클릭 시 SCR-006 이동 |
| 최근 활동 피드 | List | 최대 5건, 타임스탬프 + 작업 내용 |
| 빠른 실행 버튼 | ButtonGroup | JSON 불러오기, TC 생성, CSV 출력 단축 버튼 |

### 3.4. 사용자 흐름

1. 앱 최초 진입 시 대시보드 표시 → 모든 카드 0
2. JSON 불러오기 후 → 요구사항 카드 갱신
3. TC 생성 → TC 수 및 커버리지 카드 갱신
4. 변경 플래그 설정 → 변경 요구사항 카드 빨간색 강조

## 4. SCR-002 — 요구사항 목록

### SRS 추적: DEV_FR_001_01, DEV_FR_001_02, DEV_FR_001_03

### 4.1. 화면 목적

DEV-REQ JSON을 로드하여 계층적 그리드로 표시하고, 기본 CRUD를 지원한다.

### 4.2. 화면 구조

```
┌────────────────────────────────────────────────────────────────┐
│ 요구사항 목록                    [📂 JSON 불러오기]  [+ 추가]   │
├──────┬─────────────────────┬──────────┬────────┬──────────────┤
│req_id│ req_name            │func_class│priority│ dev_status   │
├──────┼─────────────────────┼──────────┼────────┼──────────────┤
│DEV_FR│ DEV-REQ 데이터 관리  │ 기능     │ MUST   │ Todo    [▶]  │
│_001  │                     │          │        │              │
│  ├─DEV│   JSON 파일 업로드  │ 기능     │ MUST   │ Todo    [▶]  │
│  _001 │   및 파싱          │          │        │              │
│  _01  │                     │          │        │              │
│  │ ├─D│     스키마 유효성  │ 기능     │ MUST   │ Todo         │
│  │   EV│     검사          │          │        │              │
│  │ └─D│     파싱 오류 처리 │ 기능     │ MUST   │ Todo         │

└────────────────────────────────────────────────────────────────┘
```

### 4.3. UI 요소 상세

| 요소 | 타입 | 설명 |
|------|------|------|
| JSON 불러오기 버튼 | Button (primary) | 파일 선택 다이얼로그 열기. 배경 #F97316 |
| 추가 버튼 | Button (outline) | 빈 요구사항 행 생성 |
| 계층 그리드 | TreeGrid | req_level 기반 들여쓰기 (레벨당 16px) |
| req_id 열 | text | 클릭 시 상세 패널 표시 |
| 행 컨텍스트 메뉴 | ContextMenu | 우클릭: 편집, 삭제, TC 생성, 변경 플래그 설정 |
| [▶] TC 생성 버튼 | IconButton | 해당 요구사항에 대한 LLM TC 생성 (SCR-003으로 이동) |
| 하단 상태바 | StatusBar | 로드 건수 요약, SCR-003 이동 단축 버튼 |
| 변경 플래그 아이콘 | Toggle icon | 행 앞 토글 아이콘, 설정 시 행 노란 배경 |

### 4.4. 행 스타일 규칙

| req_level | 배경색 | 폰트 |
|-----------|--------|------|
| 1 (Epic) | `#D6E4FF` | Bold 14px |
| 2 (Feature) | `#EBF4FF` | Bold 12px |
| 3+ (UserStory, Task, Sub-task) | `#FFFFFF` / 줄무늬 `#F7F9FF` | Regular 12px |
| 변경 플래그 ON | `#FFFBEB` (노란 배경) | — |

### 4.5. 파일 로드 상태

| 상태 | UI 표현 |
|------|---------|
| 로드 전 | "JSON 파일을 불러오세요" 빈 상태 안내 + 파일 업로드 영역 |
| 로드 중 | 파일명 표시 + 진행 스피너 |
| 로드 완료 | 그리드 표시 + "N건 로드됨" 성공 배너 (3초 후 자동 닫힘) |
| 오류 | 빨간 오류 배너: 파일명 + 오류 위치 + 재시도 안내 |

## 5. SCR-003 — 테스트케이스 관리

### SRS 추적: DEV_FR_002, DEV_FR_003

### 5.1. 화면 목적

요구사항 단위로 LLM을 통해 TC를 생성하고, 사용자가 Accept/Reject 및 편집을 수행한다.

### 5.2. 화면 구조 (좌우 2패널)

```
┌──────────────────────────────────────────────────────────────┐
│ 테스트케이스 관리                                              │
├────────────────────────┬─────────────────────────────────────┤
│ [왼쪽 패널]             │ [오른쪽 패널]                        │
│ 요구사항 선택           │ TC 생성 결과 / TC 목록               │
│                        │                                     │
│ ┌ 검색 ────────────┐   │ [탭] 제안 목록 | 확정 TC 목록        │
│ │🔍 요구사항 검색   │   │                                     │
│ └──────────────────┘   │ [▶ LLM TC 생성]  [⟳ 재생성]         │
│                        │                                     │
│ ○ DEV_FR_001_01        │ ── 제안 목록 ──────────────────────  │
│   JSON 파일 업로드      │                                     │
│ ○ DEV_FR_001_02        │ [Normal] JSON 파일 업로드 성공 시     │
│   그리드 표시           │   title: 요구사항 목록이 그리드에…    │
│ ○ DEV_FR_002_02 ●      │   given: 앱 실행 중이고 JSON…         │
│   (선택됨)              │   when: JSON 파일 선택               │
│                        │   then: 목록 표시됨                  │
│ [전체 선택]  [생성 →]   │   [✓ Accept] [✗ Reject]            │
│                        │                                     │
│                        │ [Exception] 스키마 오류 파일 업로드  │
│                        │   ...                               │
│                        │   [✓ Accept] [✗ Reject]            │
│                        │                                     │
│                        │ [Boundary] 빈 파일 업로드 시         │
│                        │   ...                               │
│                        │   [✓ Accept] [✗ Reject]            │
└────────────────────────┴─────────────────────────────────────┘
```

```

│ [탭] 확정 TC 목록                           [+ TC 추가]       │
├─────────┬───────────────┬────────┬─────────────────┬────────┤
│ tc_id   │ title         │ type   │ expected_result  │ status │
├─────────┼───────────────┼────────┼─────────────────┼────────┤
│TC-...001│[인라인 편집가능]│Normal │[인라인 편집가능] │Accepted│
│TC-...002│...            │Exception│...             │Accepted│
│         │               │        │                 │  [🗑]  │
└─────────┴───────────────┴────────┴─────────────────┴────────┘
```

### 5.3. UI 요소 상세

| 요소 | 타입 | 설명 |
|------|------|------|

## 6. SCR-004 — 추적 매트릭스

### SRS 추적: DEV_FR_004_01, DEV_FR_004_01_01, DEV_FR_004_02

### 6.1. 화면 목적

요구사항↔TC 매핑 관계를 테이블로 시각화하고 편집한다. 커버리지를 산출하여 표시한다.

### 6.2. 화면 구조

```
┌──────────────────────────────────────────────────────────────┐
│ 추적 매트릭스               커버리지: [██████░░] 65% (N/M건)  │
├──────────────┬─────────┬─────────┬─────────┬────────────────┤
│ 요구사항 ↓   │TC-001   │TC-002   │TC-003   │ 커버리지       │
│ TC → 　     │(Normal) │(Exception│(Boundary│                │
├──────────────┼─────────┼─────────┼─────────┼────────────────┤
│DEV_FR_001_01 │  ✓ 🔲   │  ✓ 🔲   │  ✓ 🔲   │  3/3  100%     │
│DEV_FR_001_02 │  ✓ 🔲   │         │         │  1/3   33%     │
│DEV_FR_002_02 │         │  ✓ 🔲   │  ✓ 🔲   │  2/3   67%     │
│DEV_FR_003_01 │         │         │         │  0/3    0% ⚠   │
├──────────────┼─────────┼─────────┼─────────┼────────────────┤
│ TC 커버 요구사항│  2건   │  2건    │  2건    │                │
└──────────────┴─────────┴─────────┴─────────┴────────────────┘
```

*✓ = 매핑됨 / 🔲 = 클릭 시 매핑 제거 / 빈 셀 클릭 시 매핑 추가*

### 6.3. UI 요소 상세

| 요소 | 타입 | 설명 |
|------|------|------|
| 커버리지 프로그레스바 | ProgressBar | #F97316 배경, 퍼센트 텍스트 동시 표시 |
| 매트릭스 셀 | ToggleCell | 클릭 시 매핑 추가/제거 토글 |
| 매핑 체크 아이콘 | CheckIcon | #1CB357 (초록) |
| 0% 커버리지 행 | 경고 강조 | 배경 `#FEF2F2`, 아이콘 ⚠ 표시 |
| 행 필터 | Dropdown | 커버리지 0%, MUST만 등 필터링 |
| 열 정렬 | Sort | TC ID 또는 시나리오 유형 기준 정렬 |

### 6.4. 인터랙션
